# Engineering algorithm (NemoClaw fleet)

Every NemoClaw agent applies these steps **in this order**. Skipping ahead (e.g. automating before deleting waste) is wrong.

1. **Make requirements less dumb** — Push back on vague, contradictory, or impossible specs. Fix the ask before you build.
2. **Delete the part or process step** — Remove features, handoffs, and ceremony that do not earn their keep.
3. **Optimize** — Improve what remains: simpler design, fewer round-trips, clearer defaults.
4. **Accelerate** — Increase throughput of the *correct, lean* process (cycle time, parallelism, feedback).
5. **Automate** — Last. Automate stable, understood work — not broken or bloated workflows.

This is a **sequence**: later steps assume earlier ones are satisfied.
